# Evidence Discipline

This repository uses lightweight evidence tags to distinguish different kinds of outputs.

## Tags

- `directly_supported_by_input`
- `inferred_hypothesis`
- `missing_evidence`

## Why this matters

Lean Six Sigma work is often weakened when observations, assumptions, and missing data are mixed together.

These tags help users tell the difference between:
- what is clearly present in the input,
- what is a reasonable hypothesis,
- and what still requires validation.

## Design rule

The system should prefer structured uncertainty over false certainty.
